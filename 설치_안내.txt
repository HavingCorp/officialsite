HAVING - 국문 SEO 패키지
========================================
국내 검색(네이버/구글) 대응을 위해 6개 핵심 페이지의
국문 전용 버전을 추가합니다.

배경: 기존 사이트는 JS로 언어를 전환해서, 검색엔진이
      영문 HTML만 읽어 한글 검색에 노출되지 않았습니다.
      이제 한글이 처음부터 박힌 국문 페이지를 따로 두어
      네이버/구글이 한국어 페이지로 색인할 수 있습니다.


[A. 올릴 파일 - 신규 6개]
--------------------------------------
  index-ko.html          홈 (국문)
  ingredients-ko.html    원료 (국문)   ※ 아래 [D] 참고
  finished-ko.html       완제품 (국문)
  about-ko.html          회사소개 (국문)
  fragrance-ko.html      향료 (국문)
  inquiry-ko.html        문의 (국문)
  scripts/build-ko-pages.py   국문 생성 스크립트 (재생성용, 선택)


[B. 덮어쓸 파일 - 수정 9개]
--------------------------------------
  index.html  about.html  finished.html
  fragrance.html  inquiry.html  ingredients.html
     → 각 영문 페이지에 hreflang(영↔국 연결) 태그 추가
     → 언어 버튼(KO) 클릭 시 국문 페이지로 이동하도록 연결

  seo.config.json   국문 6개 등록 + 삭제된 고아파일 정리
  sitemap.xml       국문 6개 URL 추가 (총 16개)


[C. 저장소에서 삭제할 파일 - 7개]
--------------------------------------
  README.txt
  이미지변환/  (폴더 전체)
  ingredients-en.html
  ingredients-en.json
  ingredients-ko.json   ※ 루트에 있던 중복 사본.
                          data/ingredients-ko.json 은 실사용이니 두세요!
  (ingredients-ko.html 은 삭제 아님 - 아래 [D] 참고)

  이유: 어디서도 링크되지 않는 구버전/작업용 파일들입니다.
        공개 저장소에 있으면 중복 색인 위험 + 불필요 노출.


[D. ingredients-ko.html 주의]
--------------------------------------
  기존에 같은 이름의 '구버전 고아 파일'이 있었습니다.
  이번에 그 자리에 '새 국문 원료 페이지'를 덮어씁니다.
  → [A]의 ingredients-ko.html 로 그냥 덮어쓰면 됩니다.
     (구버전은 자동으로 대체됨)


[검증 완료]
--------------------------------------
  · SEO 검사 329개 전부 통과 (0 실패)
  · 국문 6개: 한글이 HTML에 직접 박힘 (JS 없이 크롤러가 읽음)
  · hreflang 영↔국 6쌍 상호 연결 확인
  · lang="ko", 국문 title/description 적용
  · 언어 버튼으로 영↔국 이동 정상
  · 원료 테이블은 기존 JS가 그대로 처리 (건드리지 않음)


[배포 후 - 네이버/구글]
--------------------------------------
  · 네이버 서치어드바이저 → 사이트맵 재제출 (sitemap.xml)
  · 구글 서치콘솔 → 국문 주요 URL 색인 요청
      having.co.kr/index-ko.html
      having.co.kr/ingredients-ko.html
      having.co.kr/finished-ko.html
  · 색인/노출까지 보통 수일~수주 소요
